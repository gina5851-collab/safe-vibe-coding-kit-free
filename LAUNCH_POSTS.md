## Reddit-Style Launch Post

**Title:** I made a guardrail kit for non-coders using Cursor / Claude Code

Like a lot of people, I build with Cursor and Claude Code without being a professional developer. The tools are genuinely powerful, but I kept running into the same failure modes:

- vague prompt → agent touches unrelated files
- agent runs a refactor nobody asked for
- package.json or lockfiles get edited
- git push or deploy happens too early
- the agent says “Done” but I have no clear report of what changed

So I packaged the rules and prompts I use into a small Markdown kit:

- CLAUDE.md template
- Cursor rules template
- no-touch files rule
- no-push / no-deploy rule
- plan-before-editing prompt
- completion report template
- emergency recovery prompt
- pre-deploy checklist

This is not a magic prompt pack and it does not guarantee that agents never make mistakes. It’s a practical guardrail layer: the agent proposes, the human approves, and dangerous commands don’t run automatically.

I also published a free sample repo with three core files.

Full kit is $9 on Gumroad:
